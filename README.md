## 评价
**披着 OS 外壳的 Linux 杂货铺**：这是一个典型的“低配版”工程师培养案例，它略过了底层架构的厚度，只给了你一层运维的皮毛。你学了一堆可能会随版本更新而过时的 Linux 命令，却没能建立起对硬件和软件之间那层“透明契约”的深刻认知。

### 1️⃣ OS / 计算机网络课程：**以“自救”为前提**

这些课通常具备以下特征之一或多个：

* 多名老师，各讲各的，毫无相互联系协调
* PPT 极简,某门课td答案全靠课上自己收集
* 考试内容和授课内容错位

#### 生存策略：

* ❗ **第一周就找“往年卷 / 样题”**
* ❗ **以考试为导向反推学习内容**
* ❗ **不要假设“老师讲的就是要考的”**
* ❗ **遇到矛盾内容，以出卷老师为准**

### TD / Exercise：不要期待“教学质量”

* **有 TD ≠ TD 有用**
* **有答疑 ≠ 能解答你的问题**
